## 1. Test Scope

**In scope:**
- HTML output structural integrity: `#overview` section present, contains `.itinerary-table`, appears before day cards.
- HTML output structural integrity: `#budget` section present, contains budget content, appears after day cards.
- Document assembly order: overview → days (chronological) → budget, verified via DOM ordering.
- Section isolation: `#overview` and `#budget` are not wrapped inside `.day-card` elements.
- Fragment file existence: `fragment_overview_LANG.html` and `fragment_budget_LANG.html` present in trip folder after render.
- No duplicate or missing content due to the pipeline change (content byte-equivalence guard via existing regression checks).

**Out of scope:**
- Internal pipeline execution order (sequential vs. parallel subagent spawning) — this is a rule-file behavioral directive not observable in the rendered HTML.
- `rendering-config.md` and `SKILL.md` rule-text content validation — rule files are not test artifacts; the rendered output is the contract.
- Incremental rebuild execution path — BRD REQ-006 explicitly states incremental mode is unchanged; no new test coverage needed.
- Shell fragment content (PAGE_TITLE, NAV_LINKS, NAV_PILLS) — these are untouched by this change and already covered by `structure.spec.ts` and `navigation.spec.ts`.
- Visual regression snapshots — no HTML structure change means snapshot diffs are irrelevant.

**Test type:** Regression (output preservation) — no new progression tests are required because the HTML output is structurally identical before and after. The existing regression suite exercises the same DOM assertions. New tests cover only the assembly-order guarantee and fragment-file existence, which were not previously asserted.

### TC-005: Assembly order — overview precedes first day card, budget follows last day card

**Traces to:** REQ-005 → AC-1, REQ-005 → AC-3
**Type:** Regression
**Spec file:** `overview-budget.spec.ts` (new `test()` block in new `test.describe('Assembly Order', ...)`)
**Priority:** Critical

**Preconditions:**
- `trip_full_LANG.html` loaded via shared-page fixture.
- `tripConfig.dayCount` available for determining first and last day indices.

**Steps:**
1. Call `page.evaluate()` that returns `{ overviewBeforeDay0: boolean, budgetAfterLastDay: boolean }` using `Node.compareDocumentPosition()` internally.
2. Assert `overviewBeforeDay0` is `true` using `expect.soft()` with message `'#overview must precede #day-0 in document order'`.
3. Assert `budgetAfterLastDay` is `true` using `expect.soft()` with message `'#budget must follow #day-{N} in document order'`.

**Expected result:**
- `#overview` DOM position precedes `#day-0`.
- `#budget` DOM position follows `#day-{N}`.
- Concatenation order is: overview → days → budget.

**Implementation notes:**
- Use `page.evaluate()` to access `Node.compareDocumentPosition()` — not directly accessible via Playwright locators.
- The evaluate callback must return a named-property object `{ overviewBeforeDay0: boolean, budgetAfterLastDay: boolean }` (not a single compound boolean) so each `expect.soft()` can produce a distinct, actionable failure message.
- Day index for last day: `tripConfig.dayCount - 1`.
- Shared-page fixture — no mutation.
- This is the primary regression guard for REQ-005 AC-3 (order unchanged).

### TC-008: Fragment files exist on disk after render — overview and budget

**Traces to:** REQ-004 → AC-1
**Type:** Regression
**Spec file:** `overview-budget.spec.ts` (new `test()` block in new `test.describe('Fragment File Existence', ...)`) — uses Node.js `fs`, not Playwright DOM
**Priority:** High

**Preconditions:**
- Render pipeline has completed (trip folder contains generated files).
- `getManifestPath()` (from `tests/utils/trip-folder.ts`) used to locate trip folder.

**Steps:**
1. Derive trip folder path from manifest path (strip `manifest.json` filename).
2. Derive language code from `tripConfig.labels.langCode`.
3. Construct expected paths using `path.join()`: `fragment_overview_{LANG}.html` and `fragment_budget_{LANG}.html`.
4. Assert both files exist using `fs.existsSync()`, each as a separate `expect.soft()` with a descriptive message.

**Expected result:**
- `fragment_overview_{LANG}.html` exists in the trip folder.
- `fragment_budget_{LANG}.html` exists in the trip folder.

**Implementation notes:**
- This is a filesystem-level check, not a DOM test. The `test.describe('Fragment File Existence', ...)` block uses `import { test as baseTest, expect as baseExpect } from '@playwright/test'` — this is the **only** prescribed import strategy. No alternative import approach is permitted.
- The spec file's top-level `test` and `expect` imports remain from `tests/fixtures/shared-page.ts` for all DOM test blocks (TC-001 through TC-007). The `baseTest`/`baseExpect` aliases are used exclusively within the "Fragment File Existence" describe block.
- This is a well-established Playwright pattern: a `test.describe` block using the base `@playwright/test` import can coexist in a file that imports a custom fixture, provided the two are never mixed within the same describe block.
- No Playwright page navigation needed; TC-008 does not use the shared-page fixture.
- Language code sourced from `tripConfig.labels.langCode` — language-independent per automation rule §7.1.
- Trip folder path derived from `getManifestPath()` — trip-independent per automation rule §7.2.
- Path construction must use `path.join(tripFolderPath, 'fragment_overview_' + langCode + '.html')` — not hardcoded string separators — to remain platform-agnostic per automation rule §7.2.

## 6. Risk & Mitigation

| Risk | Mitigation |
|---|---|
| `Node.compareDocumentPosition()` in TC-005 behaves unexpectedly across Playwright page evaluation contexts | Use a `page.evaluate()` call that returns a named-property object `{ overviewBeforeDay0: boolean, budgetAfterLastDay: boolean }` — each boolean is computed inside the callback so the raw bitmask never crosses the serialization boundary |
| TC-008 filesystem check is tightly coupled to trip folder path convention | Use `getManifestPath()` from the existing `trip-folder.ts` utility (already used in `progression.spec.ts`) — avoids hardcoded paths |
| Fragment filename uses language code suffix; if `langCode` changes, TC-008 path construction breaks | `tripConfig.labels.langCode` is the single source of truth — no hardcoded suffix in test code |
| TC-005 uses `dayCount - 1` as last day index; if arrival/departure day structure changes, the last day ID may shift | Use `tripConfig.dayCount - 1` as the last day index consistently with how `progression.spec.ts` defines `lastActiveDay` — if structure changes, both tests are affected equally and the rule is clear |
| Adding TC-004 assertion, TC-005 describe block, and TC-008 describe block to `overview-budget.spec.ts` increases file length | These are naturally grouped (overview/budget structural checks) — no spec-splitting needed; the file remains focused on a single concern |
| TC-008 uses `baseTest`/`baseExpect` imports from `@playwright/test` alongside the shared-page fixture import | Scope TC-008 strictly within `test.describe('Fragment File Existence', () => { ... })` using only `baseTest`/`baseExpect` inside that block. All other describe blocks use the shared-page fixture imports exclusively. Playwright resolves fixture scope at describe-block level — mixing imports is safe when each describe block uses one import source consistently |
